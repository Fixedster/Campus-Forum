# Campus Forum 校园技术论坛系统 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景

Campus Forum 是一个面向高校学生的技术交流社区平台，旨在为校园开发者提供一个分享技术知识、交流开发经验、记录学习成长的平台。项目基于 Spring Boot + React 全栈技术构建，采用现代化的前后端分离架构。

### 1.2 项目目标

- 为高校学生提供专业的技术文章发布与阅读平台
- 支持多维度内容分类与标签管理
- 实现用户间的互动社交功能（点赞、评论、收藏、关注）
- 提供完善的后台管理系统，支持内容审核与用户管理
- 确保系统安全性，采用 JWT 双 Token 认证机制
- 提供完善的 AI 智能助手（对话、写作辅助、图像生成）

### 1.3 目标用户

| 用户类型 | 角色描述 | 权限范围 |
|---------|---------|---------|
| 超级管理员 | 系统最高管理者 | 全部功能 |
| 管理员 | 内容审核与用户管理 | 内容管理、用户管理 |
| 普通用户 | 内容创作者与消费者 | 发布文章、互动功能 |

## 4. 功能规格

### 4.1 用户模块

#### 4.1.1 用户注册
- **功能描述**：新用户注册账号
- **输入参数**：用户名、密码、邮箱（选填）、昵称（选填）
- **业务规则**：
  - 用户名长度 3-20 位，唯一
  - 密码长度 6-32 位
  - 默认角色为普通用户（role=0）
  - 密码使用 BCrypt 加密存储
- **返回结果**：JWT 双 Token

#### 4.1.2 用户登录
- **功能描述**：用户登录系统
- **输入参数**：用户名、密码
- **业务规则**：
  - 验证用户名和密码
  - 验证用户状态是否正常
  - 验证密码是否正确
- **返回结果**：JWT 双 Token（Access Token + Refresh Token）

#### 4.1.3 Token 刷新
- **功能描述**：刷新过期 Access Token
- **输入参数**：Refresh Token
- **业务规则**：
  - Refresh Token 有效期 7 天
  - Access Token 有效期 2 小时
  - Refresh Token 只能使用一次
- **返回结果**：新的 JWT 双 Token

#### 4.1.4 用户登出
- **功能描述**：用户登出系统
- **业务规则**：将 Token 加入 Redis 黑名单
- **返回结果**：成功/失败

#### 4.1.5 获取用户信息
- **功能描述**：获取指定用户或当前用户信息
- **业务规则**：返回用户公开信息（不含密码）
- **返回结果**：UserVO（用户视图对象）

#### 4.1.6 更新个人资料
- **功能描述**：更新当前用户资料
- **输入参数**：昵称、邮箱、简介、学院、专业
- **业务规则**：仅能更新当前登录用户
- **返回结果**：更新后的用户信息

#### 4.1.7 关注/取消关注用户
- **功能描述**：关注或取消关注其他用户
- **业务规则**：不能关注自己
- **返回结果**：成功/失败

### 4.2 文章模块

#### 4.2.1 发布文章
- **功能描述**：发布新文章
- **输入参数**：标题、内容、摘要（选填）、分类ID、标签（选填）、来源
- **业务规则**：
  - 标题长度 5-120 位
  - 内容支持 Markdown 格式
  - 自动生成内容摘要（取前200字符）
  - 默认状态为已发布
- **返回结果**：文章ID

#### 4.2.2 更新文章
- **功能描述**：更新已发布的文章
- **输入参数**：文章ID、标题、内容、摘要、分类ID、标签
- **业务规则**：
  - 仅作者或管理员可更新
  - 更新后文章状态保持不变
- **返回结果**：成功/失败

#### 4.2.3 删除文章
- **功能描述**：删除文章（软删除）
- **输入参数**：文章ID
- **业务规则**：
  - 仅作者或管理员可删除
  - 执行逻辑删除
- **返回结果**：成功/失败

#### 4.2.4 文章列表
- **功能描述**：获取文章列表（支持分页、筛选、排序）
- **输入参数**：
  - keyword：关键词搜索
  - categoryId：分类ID
  - tagId：标签ID
  - sort：排序方式（latest/hot/like）
  - page：页码
  - size：每页数量
- **返回结果**：分页后的文章列表

#### 4.2.5 文章详情
- **功能描述**：获取文章详情
- **输入参数**：文章ID
- **业务规则**：
  - 浏览数 +1
  - 返回文章完整信息及作者信息
- **返回结果**：文章详情

#### 4.2.6 点赞文章
- **功能描述**：点赞或取消点赞文章
- **业务规则**：
  - 同一用户对同一文章只能点赞一次
  - 取消点赞后点赞数 -1
  - 点赞后点赞数 +1
- **返回结果**：当前点赞状态

#### 4.2.7 收藏文章
- **功能描述**：收藏或取消收藏文章
- **业务规则**：
  - 同一用户对同一文章只能收藏一次
  - 取消收藏后收藏数 -1
  - 收藏后收藏数 +1
- **返回结果**：当前收藏状态

#### 4.2.8 获取用户文章列表
- **功能描述**：获取指定用户的文章列表
- **输入参数**：用户ID、页码、每页数量
- **返回结果**：该用户的文章列表

#### 4.2.9 获取用户收藏列表
- **功能描述**：获取当前用户的收藏列表
- **输入参数**：页码、每页数量
- **返回结果**：收藏的文章列表

#### 4.2.10 管理功能
| 功能 | 权限 | 说明 |
|------|------|------|
| 置顶文章 | 管理员 | 设置/取消文章置顶 |
| 加精文章 | 管理员 | 设置/取消文章加精 |
| 下架文章 | 管理员 | 将文章状态设为已下架 |

### 4.3 评论模块

#### 4.3.1 获取文章评论
- **功能描述**：获取文章的所有评论
- **输入参数**：文章ID
- **业务规则**：
  - 支持嵌套评论（最多2层）

- **返回结果**：评论列表（包含回复）

- **业务规则**：
  - 评论长度不超过 1000 字符
  - 回复评论时需要指定父评论ID
  - 发送评论通知给文章作者
- **返回结果**：评论ID

#### 4.3.3 删除评论
- **功能描述**：删除评论（软删除）
- **输入参数**：评论ID
- **业务规则**：
  - 仅评论者或管理员可删除
  - 执行逻辑删除
- **返回结果**：成功/失败

### 4.5 AI 助手模块

#### 4.5.1 AI 智能对话
- **功能描述**：用户与 AI 助手进行对话交流
- **输入参数**：用户输入的消息内容
- **业务规则**：
  - 需要用户登录后才能使用
  - 支持多轮对话上下文记忆
  - 使用 DeepSeek 大模型（通过 LangChain4j 封装 OpenAI 兼容接口）
  - 支持清空对话和复制回复内容
- **返回结果**：AI 回复内容

#### 4.5.2 AI 写作助手
- **功能描述**：根据用户输入的主题生成 Markdown 技术文章
- **输入参数**：主题、文章类型（选填）
- **业务规则**：
  - 需要用户登录后才能使用
  - 生成内容为 Markdown 格式，可直接复制到发布页面使用
- **返回结果**：Markdown 格式的文章内容

#### 4.5.3 AI 图像生成
- **功能描述**：根据文字描述生成图片
- **输入参数**：图片描述文本
- **业务规则**：
  - 需要用户登录后才能使用
  - 使用 DALL-E 模型生成图片
  - 图片尺寸为 256x256
- **返回结果**：生成的图片 URL

#### 4.5.4 对话管理
- **功能描述**：管理 AI 对话
- **业务规则**：
  - 支持清空当前对话
  - 支持复制 AI 回复内容
- **返回结果**：操作成功/失败

## 6. 安全规格

### 6.1 认证机制

#### JWT 双 Token 机制
- **Access Token**：
  - 有效期：2 小时
  - 用途：API 接口访问认证
  - 存储：前端内存（避免 XSS 攻击）

- **Refresh Token**：
  - 有效期：7 天
  - 用途：刷新 Access Token
  - 存储：HttpOnly Cookie 或本地存储

#### Token 黑名单
- 用户登出时，将 Token 加入 Redis 黑名单
- 黑名单过期时间与 Token 过期时间一致
- 验证 Token 时检查是否在黑名单中

### 6.2 权限控制

#### 角色权限
| 角色 | 值 | 权限 |
|------|---|------|
| 普通用户 | 0 | 发布文章、评论、互动 |
| 管理员 | 1 | + 内容管理、用户管理 |
| 超级管理员 | 2 | + 系统配置管理 |

#### 权限注解
- `@RequireLogin`：需要登录
- `@RequireRole(1)`：需要管理员权限
- `@RequireRole(2)`：需要超级管理员权限

### 6.3 数据安全

- 密码使用 BCrypt 加密存储
- SQL 注入防护：使用 MyBatis-Plus 参数绑定
- XSS 防护：内容转义处理
- CSRF 防护：Token 验证

## 10. AI 配置说明

### 10.1 配置文件
AI 功能基于 LangChain4j 封装，默认对接 DeepSeek（兼容 OpenAI 接口），在 `application.yml` 中配置：

```yaml
ai:
  openai:
    api-key: sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    base-url: https://api.deepseek.com
    model-name: deepseek-chat
    image-model-name: dall-e-2
    image-size: 256x256
    enabled: true
```

### 10.2 配置项说明
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| api-key | DeepSeek/OpenAI API 密钥 | - |
| base-url | API 基础地址 | https://api.deepseek.com |
| model-name | 对话模型名称 | deepseek-chat |
| image-model-name | 图像生成模型 | dall-e-2 |
| image-size | 生成图片尺寸 | 256x256 |
| enabled | 是否启用 AI 功能 | true |

### 10.3 优雅降级
AI 服务通过 `@ConditionalOnProperty` 注解控制，当 `ai.openai.enabled=false` 或未配置 API 密钥时，AI 功能自动禁用，系统其余功能正常运行不受影响。
